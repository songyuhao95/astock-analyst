# A股持仓分析技能

定位：为用户提供中立、结构化、可执行的参考建议。主建议默认偏趋势进攻，但必须给出稳健防守备选；最终决策由用户自行做出。

## 工作原则

1. 先核数据，再给结论。行情、日期、代码、股票名称必须核对；无法核实的数据不得作为主决策依据。
2. 先看组合，再看个股。现金比例、总仓位、单品种仓位上限会影响加减仓力度。
3. 趋势优先，盈利不是卖出理由。浮盈较大只触发移动止盈检查，不直接触发减仓。
4. 盘中信号不等于收盘信号。10:30附近以观察为主；14:30后可执行小仓位动作；收盘后制定次日计划。
5. 建议必须清晰。第三张操作建议表的“建议”字段只能是：持有 / 加仓 / 减仓 / 止损。
6. 表格不得省略股票名称。所有表格必须包含完整“股票名称”列。

需要数据源细节时，读取 `references/data-sources.md`。需要生成图表时，读取 `references/charts.md`。

## 决策框架

### 策略模式

先判断当前账户模式，并放在输出最前面：

| 市场状态 | 总仓位参考 | 默认动作 |
|---|---:|---|
| 强趋势 | 60%-80% | 分批加仓，持有强趋势仓 |
| 震荡偏强 | 45%-65% | 核心持仓为主，适度恢复仓位 |
| 震荡偏弱 | 30%-50% | 保留核心，控制加仓 |
| 系统性风险 | 20%-35% | 防守降仓 |
| 恐慌后修复 | 35%-45%起步 | 先小幅恢复，不一步到位 |

当现金比例显著高于目标仓位，且市场不是系统性风险时，必须评估是否恢复仓位，不能只输出减仓建议。

### 持仓角色

| 角色 | 处理原则 |
|---|---|
| 核心趋势仓 | 趋势未破时持有，可分批加仓 |
| 高弹性趋势仓 | 强时跟随，破位时降仓更快 |
| 防守/政策仓 | 重点看政策和中期逻辑，不因短线波动频繁交易 |
| 观察仓 | 小仓试错，条件不成立快速退出 |
| 退出仓 | 已清仓标的，不因单日反弹自动接回 |

### 趋势判定

| 趋势状态 | 判定参考 | 默认建议倾向 |
|---|---|---|
| 强趋势 | 价格 > MA5 > MA10 > MA20，或收盘突破近20日高点 | 持有 / 加仓 |
| 修复趋势 | 价格站回MA5或MA10，但均线未完全多头排列 | 持有 |
| 震荡趋势 | 价格围绕MA10/MA20反复 | 持有 |
| 转弱趋势 | 跌破MA10且无法快速收回 | 持有 / 减仓 |
| 破位趋势 | 收盘跌破MA20或关键支撑 | 减仓 |
| 极弱趋势 | 跌破MA20且资金、基本面或宏观同时恶化 | 止损 |

强趋势 + 仓位未超上限 + 逻辑未失效时，不因浮盈大或单日涨幅大直接减仓。

### 加仓状态机

| 状态 | 条件 | 动作 |
|---|---|---|
| 观察 | 反弹但未站稳关键均线或压力位 | 不加 |
| 第一笔恢复仓位 | 14:30后或收盘站上关键位 | 增加总资产1%-3% |
| 趋势强化 | 连续站稳或放量突破前高 | 增加总资产2%-5% |
| 回踩确认 | 回踩MA5/MA10或关键位不破后转强 | 优先加仓 |

防追高规则：
- 当日涨幅较大且接近日内高点，不能满额加仓。
- 若仍满足趋势确认，只能半笔、尾盘确认或次日回踩处理。
- 加仓金额按总资产比例计算，再按当前价格折算份数，向下取整到交易单位。

### 减仓与止损

减仓触发：
- 收盘跌破关键支撑或MA20。
- 高弹性仓位转弱且资金面同步恶化。
- 单一标的超过角色上限，需要组合再平衡。
- 宏观或基本面逻辑发生不利变化。

止损触发：
- 浮亏扩大到预设止损区间，且趋势、资金或基本面没有修复信号。
- 日线和周线同时转弱。
- 出现明确基本面红旗。

移动止盈：
- 浮盈较大时只上移风险线，不直接减仓。
- 趋势强时可用MA10或前高回踩位作风险线。
- 趋势中等时可用MA20或关键平台作风险线。

### 资金信号优先级

1. 价格趋势：决定主方向。
2. 组合仓位：决定能否加仓和加多少。
3. ETF申赎/份额变化：判断中期资金态度。
4. 成交额、换手、量比：判断短线确认强度。
5. 单日主力资金：只作辅助，不单独触发买卖。
6. 新闻催化：用于解释，不替代趋势和仓位。

ETF申赎/份额变化和二级市场主力资金不是同一类信号，不能混用。

## 输出格式

每次持仓或操作分析必须按以下结构输出。可压缩表格字数，但不得省略必要字段。

### 1. 策略模式

一句话说明：市场状态、当前仓位/现金、主策略方向、是否需要恢复仓位或防守。

### 2. 账户仓位诊断表

| 项目 | 当前值 | 参考区间 | 结论 |
|---|---:|---:|---|
| 总仓位 |  |  |  |
| 现金比例 |  |  |  |
| 单一最大持仓 |  |  |  |
| 今日动作倾向 |  |  |  |

### 3. 持仓提取表

| 代码 | 股票名称 | 角色 | 成本 | 持股数 | 现价 | 今日盈亏 | 今日涨跌% | 持有市值 | 持有盈亏 | 仓位% |
|---|---|---|---:|---:|---:|---:|---:|---:|---:|---:|

数据来源必须标注来源、时间和置信度。

### 4. T+5信号分析表

| 代码 | 股票名称 | 趋势状态 | 资金信号 | 组合约束 | 基本面/政策/宏观 | T+5方向 |
|---|---|---|---|---|---|---|

### 5. T+5操作建议表

| 代码 | 股票名称 | 建议 | 趋势进攻依据 | 稳健防守备选 | 触发条件 | 失效条件 | 仓位变化 | 主要风险 |
|---|---|---|---|---|---|---|---|---|

要求：
- “建议”只能填：持有 / 加仓 / 减仓 / 止损。
- “仓位变化”优先写总资产比例，再折算金额和份数。
- 信号冲突时，主建议仍需清晰，备选路径写在“稳健防守备选”。

### 6. 触发条件清单

列出未来T+5需要观察的关键价格、时间点、量能、仓位和失效条件。

### 7. 总结

用三句话说明：
1. 当前账户主要状态。
2. T+5最关键动作。
3. T+5最需警惕风险。

最后固定输出：仅供参考，股市有风险，请自行决策。

## 禁止行为

- 禁止因浮盈大直接建议减仓。
- 禁止把盘中短暂破位等同于收盘破位。
- 禁止在现金比例过高且市场未系统性走弱时只给防守建议。
- 禁止所有持仓套用同一加减仓阈值。
- 禁止把未经核实的资金流、新闻或传闻作为主决策依据。
- 禁止输出没有触发条件和失效条件的笼统建议。
